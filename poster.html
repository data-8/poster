JupyterHub in Instruction

The Jupyter Notebook is a powerful and elegant way of creating and sharing code, visualizations, and documentation. In data science instruction, hosting notebooks in JupyterHub eliminates the need to have students replicate software installation on personal devices, reduces the demand on physical lab resources, ensures reproducibility, and more importantly, frees up time for both learning and teaching.

Notebooks

"The Jupyter Notebook is a web application that allows you to create and share documents that contain live code, equations, visualizations and explanatory text. Uses include: data cleaning and transformation, numerical simulation, statistical modeling, machine learning and much more."

- http://jupyter.org

Hub

While notebooks can be easily run on a personal device or lab machine, hosting the environment relieves the students from having to install the software themselves. It also means that each student has the same starting point and results are reproducible.

DS8 - Foundations of Data Science

The pilot course in Fall 2015 had 80 students and was deployed on intel-donated bare metal machines in EECS. In Spring 2016 we had about 450 students and deployed on Microsoft-donated Azure nodes. Both deployments were based on Jessica Hamrick's JupyterHub for COGSCI 131.

DS8 was complemented by a suite of courses in departments across campus, introducing diverse subjects through the lens of data science. These were known as "connector courses":

CEE 88 • COGSCI 88 • CS 88 • ESPM 88A • ESPM 88B • HIST 88
INFO 88 • L&S 88-1 • L&S 88-2 • STAT 88 • STAT 89a

Future Direction

  • Scaling up to support more students. We have a team of students adding statsd integration for logging and monitoring, and selenium for load testing.
  • Scaling out to support more classes. Our goal is to eventually have one JupyterHub deployment that can serve any course at UC Berkeley that wants to use Jupyter notebooks. Adding JupyterHub for a course should be as easy as creating a class web page.
  • Adding features such as Google Drive or Box integration for notebook storage and distribution, nbgrader for notebook grading, and roster-based account provisioning via SIS APIs or CalNet authz with CalGroups.
